# Skill Markdown Standard Format

All skill files must follow this exact structure. Fields marked (optional) can be omitted if not available.

```markdown
# Skill {XX}: {Name}

Cultivation: {id} - {name}
Subtitle: {subtitle}
Duration: {duration}

> {quote text} â€” {attribution}

(Quote section is optional - omit entirely if no quote)

## Overview

{overview paragraph}

## Benefits

- {benefit 1}
- {benefit 2}

## Purpose

{purpose paragraph}

## Resources

- YouTube: {url}
- SoundCloud: {url}
- PDF: {title} (optional)
- Other: {url} (optional)

## Instructions

### Step 1: {Title}

{content paragraph}

### Step 2: {Title}

{content paragraph}

(continue for all steps)

## Tips

- {tip 1}
- {tip 2}

(optional section)

## Obstacles

### {Obstacle Title}

{content paragraph}

(repeat for each obstacle, optional section)

## Insight

- Marker: {XX} - {marker name}
- Hindrance: {XX} - {hindrance name}
- Antidote: {antidote paragraph}

## Daily Application

Check-in: "{question}"

### Learnings

- {learning 1}
- {learning 2}

### Steps

1. {step 1}
2. {step 2}

(optional subsections)

## Progression

Criteria: {criteria paragraph}
Next: Skill {XX} - {name}
Previous: Skill {XX} - {name} (optional)
```

## Notes

- Use standard markdown headers (# ## ###)
- Cultivation ID format: "01", "02", etc.
- Skill ID format: "00", "01", ..., "16"
- Keep content intact, only restructure format
- Preserve all original information
